# 时间式朗读进度与跨材料导航 Plan

## 实现方案

- 后端把真实 WAV 时长作为权威数据，持久化到句子；未知时长不写库，由前端和摘要聚合按文本长度估算。
- 阅读进度保存当前句、句内秒数、倍速和朗读完成状态；句内秒数用于刷新恢复和时间跳转落点。
- 启动路径只支持上一版真实六表结构的 additive migration：迁移前复制不覆盖旧文件的数据库备份，迁移失败保留原库并拒绝启动；其他结构继续拒绝。
- 阅读页使用混合时间轴，真实时长优先，未知句按材料内已知秒/文本长度估算，播放中节流保存，关键动作立即保存。
- 材料相邻关系按 `created_at ASC, id ASC` 计算；播放器左右按钮切换较早/较晚导入材料，篇末在播放器内提示继续下一篇。

## 关键决策

- 不做全篇批量生成音频。
- 估算值不持久化，不作为权威数据。
- 书架摘要允许后端返回聚合估算，用于和阅读页保持时间进度语义一致。
- 播放器不保留显式上一句/下一句按钮；正文点击仍可定位句子。
- 已完成材料被上一篇/下一篇打开时统一定位到第一句 0 秒；播放意图只决定是否自动开始。
- 暂停拖动到未知音频句子不前台生成音频；播放中跳到未知音频句子进入加载状态，准备成功后从目标 offset 继续。
- 真实时长返回后夹紧当前 offset；播放中夹到句末则按句末结束处理。
- 拖动过程中只本地预览，释放滑块后保存最终落点；拖到 100% 或快进越过末尾等同朗读完成。
- 快退/快进不跨材料边界；跨材料切换前 best-effort 保存当前进度但不阻塞导航。
- 页面级快捷键为 `ArrowLeft` 快退 15 秒、`ArrowRight` 快进 30 秒、`Space` 播放/暂停；输入、选择、按钮、进度条和阅读设置面板内不拦截。
- 时间轴显示原始媒体时长，格式为 `MM:SS`，超过一小时用 `H:MM:SS`；估算状态只显示“约”。
- 书架主进度使用时间，句子位置作为更小更浅的次级信息。
- 时间轴/seek/格式化逻辑抽成独立 view model 并单测。

- 前端交互变更完成后用浏览器验证：播放/暂停、拖动进度条、快退 15 秒、快进 30 秒、跨材料切换、篇末继续提示。
