# Read Along Product Backlog

最后更新：2026-06-07

## 使用方式

这个 backlog 是一个持续演进的产品待办列表。每次 Sprint Planning 前，从 `P0` 中选择未完成且依赖已满足的故事进入 Sprint；完成后更新状态、验收结果和后续发现。

## 状态说明

- `Todo`：尚未开始。
- `Ready`：需求清楚，依赖明确，可以进入 Sprint。
- `Doing`：正在实现。
- `Review`：已实现，等待验证。
- `Done`：通过验收。
- `Deferred`：明确推迟，不进入 MVP。

## 优先级说明

- `P0`：MVP 必须完成，否则核心闭环不成立。
- `P1`：MVP 应完成，显著影响可用性或验收质量。
- `P2`：MVP 可选，若时间不足可顺延。
- `Later`：明确不进入第一版，仅保留为后续方向。

## MVP Goal

用户可以在 macOS 本机导入一篇可访问网页或一个文本型 PDF，将其变成结构化阅读材料，并在本地 Web 阅读器中完成句子级朗读、高亮和断点续读。

## Backlog Maintenance Rules

- 每个故事进入 Sprint 前必须处于 `Ready`。
- 每个故事必须有可验证的 Acceptance Criteria。
- 如果实现中发现范围变大，优先拆小，不直接扩展原故事。
- 如果某项需求不影响核心闭环，默认降为 `P2` 或 `Later`。
- Deferred 项不得在 MVP 中偷偷实现，除非先更新 `docs/mvp-scope.md`。
