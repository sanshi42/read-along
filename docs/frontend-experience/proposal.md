# 整体前端体验优化 Proposal

## Goal

在不改变 Read Along 核心产品能力的前提下，统一并提升书架、阅读页及其状态页面的视觉层级、交互效率、响应式体验与无障碍质量，使导入、选择材料和边听边读的主要流程更清晰、舒适且可靠。

## Boundary

### In

- 当前书架、阅读页、404 和加载、空白、错误、成功反馈状态。
- 全局设计语言、页面布局、组件视觉、响应式、键盘操作和必要的微交互。
- 当前导入、阅读偏好、句子选择、朗读控制和阅读进度反馈的前端呈现。
- 为书架恢复提示扩展阅读材料 API 摘要中的朗读位置派生数据。
- 为验证主要体验而补充必要的前端测试与浏览器验收。

### Out

- 新增材料管理、搜索、笔记、划线、统计或其他产品能力。
- 改变材料、朗读、阅读进度和阅读偏好的领域语义。
- 修改数据库结构、导入规则或 TTS 行为。
- 引入重型前端框架、组件库或复杂状态管理方案。
