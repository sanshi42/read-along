# 朗读加载流畅性 Proposal

## Goal

减少阅读材料首次朗读和连续朗读时的句间等待，让用户可以稳定地边听边读，而不是每一句都停在音频准备阶段。

目标完成后，长阅读材料也应能在本地缓存、音频生成和连续播放之间形成更顺滑的体验，并在音频准备失败时给出可恢复的状态。

## Boundary

### In

- 朗读音频的准备、生成、缓存复用和连续播放衔接体验。
- 长阅读材料的一边朗读一边准备后续音频能力。
- 句子音频状态、错误提示和重试行为。
- 保持阅读材料正文、当前句和阅读进度语义不变。

### Out

- 时间轴式整体进度、快退快进和跨阅读材料导航。
- 更改来源导入、正文抽取或材料去重规则。
- 绕过本地 TTS 或引入在线账号、同步与云端音频服务。
