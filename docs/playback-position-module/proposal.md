# 阅读进度与朗读位置 Module Proposal

## Goal

加深“阅读进度如何变成朗读位置”的 Module，让恢复、时间位置、朗读完成和保存完成状态的规则集中在更小的 Interface 后面。

目标完成后，材料库和阅读页不再各自手写同一套朗读完成与恢复语义；相关测试应直接覆盖这个 Module 的 Interface。

## Boundary

### In

- 后端朗读位置和时间式朗读位置计算的集中化。
- 前端当前句恢复锚点、朗读时间线和保存完成进度语义的集中化。
- 保持现有 API 字段、数据库 schema、阅读进度持久化格式不变。
- 增加或调整聚焦于阅读进度与朗读位置 Interface 的测试。

### Out

- 改变阅读进度数据模型或数据库结构。
- 改变播放器 UI、播放模式或跨材料导航行为。
- 引入跨语言代码生成或新的构建步骤。
- 重写音频准备、TTS 或导入流程。
