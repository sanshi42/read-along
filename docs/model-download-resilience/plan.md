# 模型下载可靠性 Plan

## 方案

- 在模型目录保存 `.part` 归档和 JSON 元数据（URL、ETag/Last-Modified、预期字节数）。
- 流式下载每个数据块，通过可选进度协议上报字节数；交互 CLI 用 Rich 渲染百分比、大小、速度与 ETA。
- 后续调用携带 `Range` 和 `If-Range`；响应版本、总大小或 Range 起点不匹配时清除局部状态并完整下载。
- 网络可恢复错误首次请求外重试 3 次；归档解压或模型校验失败时自动完整重下 1 次。
- `--restart` 仅清除未完成归档；有效模型继续复用。

## Topic 级验证

- 用伪 HTTP 响应覆盖续传、Range 回退、版本变化、短读、416、重试、完整性失败与强制重下。
- 运行 `make check`。
