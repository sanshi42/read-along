# 整体前端体验优化 Plan

## Approach

- 先通过领域文档、现有实现和真实页面审查明确产品气质、信息层级与优先使用场景，再确定统一设计系统。
- 保留现有 React、TypeScript 与原生 CSS 技术基线，使用语义化设计令牌和轻量可复用组件统一页面。
- 分别优化书架的导入与材料选择流程、阅读页的沉浸阅读与朗读控制流程；两者共享全局视觉和交互规则。
- 优先处理可访问性、主要交互可发现性、状态反馈和响应式，再处理装饰与微动效。

## Decisions

- 不新增产品能力，不改变现有阅读进度、朗读完成或自动播放语义。
- 整体采用暖纸张编辑风：保留安静、可信、沉浸的个人阅读空间气质，减少不必要的渐变、透明和阴影，通过更清晰的层级、对比和交互状态提升现代感与可用性。
- 以 macOS 桌面体验为主，窄屏完整可用；桌面布局与交互优先获得最佳体验，390px 窄屏仍可完成全部现有流程，但不为移动端建立独立产品交互体系。
- 允许重组现有页面的信息架构，但保留全部现有功能和产品边界；可重新安排导入区、材料列表、阅读页头部、阅读偏好和播放器，不新增侧栏、独立流程或新产品能力。
- 书架以选择并继续阅读已有阅读材料为首屏主任务，沿用现有材料排序；使用当前句序号和总句数展示“朗读位置”，不把它描述为阅读完成度。
- 导入使用顶部紧凑入口，点击后在书架页原位展开或收起面板；折叠时保留表单组件状态，以维持导入失败后 URL 和已选 PDF 可继续重试的现有行为。
- 阅读页正文沉浸优先，播放器使用紧凑底部控制坞；高频操作使用统一线性图标，非显然操作保留文字。
- 阅读正文使用系统宋体，界面控制使用系统黑体；浅色暖纸张与深色炭黑夜读成对设计。
- 当前句使用暖色荧光笔底纹，正在朗读时增加更强底纹和非颜色提示。
- 句子使用单一 Tab 停靠点与方向键移动焦点；阅读偏好继续使用顶栏弹出层并补齐关闭和焦点管理。
- 使用克制的功能性动效并支持减少动态效果；引入 `lucide-react` 作为统一 SVG 图标来源。

## API

- 阅读材料摘要与详情增加可选 `playback_position`，包含 `sentence_index` 与 `sentence_count`。
- 无阅读进度时返回 `null`；数据从现有当前句和句子集合派生，不修改数据库。
- 前端按全文跨度计算朗读位置：首句为 0%，末句为 100%；单句材料仅在朗读完成后显示 100%。

## Verification

- 运行 `make check`。
- 在浏览器中验证书架、阅读页、状态页面及主要交互。
- 验证浅色、深色、窄屏、键盘操作和减少动态效果场景。

## Verification Result

- `make check` 通过，覆盖 Ruff、格式检查、Pyrefly、198 项后端测试和前端生产构建。
- 浏览器验证桌面与 390px 窄屏无横向滚动；空书架自动展开导入，已有材料默认折叠且折叠控件退出键盘交互。
- 浏览器验证朗读位置、朗读完成、设置弹层焦点返回、单一句子 Tab 停靠点、方向键焦点移动和底部播放器布局。
- 浅色与深色主题的主文字、辅助文字、按钮和句子高亮对比度均达到本次设定标准；减少动态效果通过 CSS 媒体查询关闭非必要动画。
