# Read Along MVP Scope

## 目的

第一版只验证一个核心闭环：**用户提供一篇可访问的文本材料，系统把它变成可阅读、可朗读、可高亮、可断点续读的本地阅读体验**。

这个文档用于敏捷开发中的范围控制。凡是不在 “MVP 做什么” 中的内容，默认不进入第一版迭代。

## MVP 做什么

### 1. 本地个人 Web App

- 在 macOS 本机运行。
- 后端使用 Python/FastAPI。
- 前端使用 React。
- 默认只服务本机用户，不面向公网或多人协作。

### 2. 单篇材料导入

- 支持输入一个网页 URL。
- 支持上传一个文本型 PDF。
- 每次导入只处理一篇文章、一个课时或一个 PDF 文件。
- 重复导入同一 URL 时，复用已有材料或提示材料已存在。

### 3. 网页正文获取

- 使用 Scrapling 作为网页抓取和正文解析入口。
- 普通公开网页直接抓取。
- 登录态或付费网页不保存 Cookie、账号密码或导出凭据。
- 对得到这类已授权课程页，第一版采用用户手动登录的专用 Chrome 会话桥接方式读取可见正文。

### 4. PDF 文本提取

- 只支持可复制文字的文本型 PDF。
- 提取正文后按段落和句子组织。
- 保留页码或基础来源信息，方便回查。

### 5. 正文清洗与结构化

- 抽取标题、来源 URL 或文件名。
- 清理明显导航、按钮、评论区、页脚等噪声。
- 按段落切分正文。
- 按句子切分段落，并生成稳定的 `sentence_id`。
- 保留正文原意，不使用 LLM 改写或总结。

### 6. 本地朗读

- 默认使用 macOS `say` 作为第一版 TTS。
- 按句子预生成音频文件。
- 音频文件存入本地私有缓存。
- TTS 设计为适配器接口，后续可替换为 Piper、Kokoro、Coqui 等本地神经 TTS。

### 7. 阅读器交互

- 展示舒适的正文阅读界面。
- 支持字号调整。
- 支持行距调整。
- 支持明暗主题。
- 支持播放、暂停、上一句、下一句。
- 支持倍速播放。
- 当前朗读句子高亮。
- 播放到句末后自动进入下一句。

### 8. 本地进度保存

- 保存每篇材料的当前阅读句子。
- 保存最近播放位置。
- 刷新页面或重启服务后能恢复进度。
- 已导入正文和已生成音频在断网状态下仍可使用。

## MVP 不做什么

### 1. 不做批量课程抓取

- 不输入课程主页后自动导入整门课程。
- 不自动翻页。
- 不自动遍历目录。
- 不做抓取队列、限速策略、批量失败恢复。

### 2. 不绕过访问权限

- 不破解登录。
- 不绕过付费限制。
- 不保存或导出账号密码。
- 不保存或导入浏览器 Cookie 文件。
- 不提供分享课程内容的功能。

### 3. 不做扫描版 PDF OCR

- 不处理图片扫描书。
- 不做版面识别。
- 不做 OCR 校对。

### 4. 不做高级学习功能

- 不做 LLM 总结。
- 不做 LLM 改写口播稿。
- 不做自动出题。
- 不做知识卡片。
- 不做学习报告。
- 不做笔记、划线、标签、全文搜索。

### 5. 不做多端同步

- 不做账号系统。
- 不做云端存储。
- 不做手机 App。
- 不做局域网多设备访问的权限控制。

### 6. 不做生产级部署

- 不做 Docker 部署。
- 不做 HTTPS。
- 不做反向代理。
- 不做监控告警。
- 不做多用户隔离。

## MVP 用户故事

- 作为学习者，我可以输入一篇网页链接，将它导入为本地阅读材料。
- 作为学习者，我可以上传一个文本型 PDF，将它导入为本地阅读材料。
- 作为学习者，我可以在阅读器中调整字号、行距和主题。
- 作为学习者，我可以点击播放，让系统从当前句开始朗读。
- 作为学习者，我可以看到当前朗读句子被高亮。
- 作为学习者，我可以暂停、切换上一句/下一句、调整倍速。
- 作为学习者，我关闭页面后再次打开，可以继续上次的位置。

## MVP 验收标准

- 可以导入至少一个公开网页，并生成可阅读正文。
- 可以导入一个文本型 PDF，并按段落/句子展示。
- 可以通过专用 Chrome 会话桥接导入一个用户已授权访问的得到单篇课程页。
- 正文区域不包含明显评论区、导航、登录按钮或页脚噪声。
- 点击播放后，当前句子被高亮且音频正常播放。
- 播放结束后能自动进入下一句。
- 刷新页面后能恢复最近阅读位置。
- 后端重启后材料列表、正文结构和进度仍存在。
- `say` 不可用或句子音频生成失败时，界面给出明确错误。

## 第一版迭代建议

### Sprint 1：本地材料库与正文结构

- 建立 FastAPI 服务和本地存储。
- 实现材料模型、段落模型、句子模型和进度模型。
- 实现文本型 PDF 导入。
- 实现基础材料列表和阅读页。

### Sprint 2：网页导入与清洗

- 接入 Scrapling。
- 支持公开网页导入。
- 支持专用 Chrome 会话桥接导入得到单篇文章。
- 完成正文清洗规则和重复导入处理。

### Sprint 3：TTS 与播放器

- 实现 macOS `say` TTS 适配器。
- 按句生成音频缓存。
- 实现前端播放器、句子级高亮、倍速和进度保存。

### Sprint 4：体验打磨与验收

- 完成字号、行距、明暗主题。
- 补齐错误提示和空状态。
- 覆盖 MVP 验收测试。
- 用目标得到课程页和至少一个文本型 PDF 做端到端验证。

## 决策记录

- 产品边界：个人本地工具。
- 平台边界：macOS 优先。
- 界面形态：本地 Web App。
- 技术栈：FastAPI + React。
- 网页导入：Scrapling + 专用 Chrome 会话桥接。
- PDF 范围：文本型 PDF。
- TTS 路线：先 macOS `say`，后续可插拔本地神经 TTS。
- 高亮粒度：句子级。
- LLM 角色：第一版不改写正文。
- 存储策略：本地私有库。
