# Read Along

Read Along 把用户可访问的单篇来源内容转成忠实、可朗读、可断点续读的本地阅读材料。这个上下文关注阅读材料从导入到本地保存和阅读的完整生命周期。

## Language

**阅读材料（Reading Material）**:
材料库中的一篇可阅读内容，包含来源身份、标题、结构化正文、句子音频状态和阅读进度。
_Avoid_: 文章、文档、资源

**结构化正文（Structured Body）**:
从来源内容清洗得到、按顺序组织为段落和句子的忠实正文。它是正文等价性判断和 `content_hash` 的依据。
_Avoid_: 原始文本、提取文本

**材料库（Material Library）**:
保存阅读材料的本地私有集合，也指拥有阅读材料身份、重复判断、原子持久化、完整读取和删除生命周期的领域概念。
_Avoid_: Repository、数据库、存储层

**来源身份（Source Identity）**:
一篇阅读材料的一个导入来源，由来源类型和稳定来源键标识，并保留来源 URI 用于展示和回查。一篇阅读材料可以关联多个来源身份。
_Avoid_: 来源字段、链接记录

**当前句（Current Sentence）**:
用户在阅读材料中当前定位、准备继续阅读或正在朗读的句子。它可以由手动选择或连续朗读改变，不等同于最后完整播放的句子。
_Avoid_: 播放句、已读句

**阅读进度（Reading Progress）**:
一篇阅读材料中当前句、播放倍速偏好与朗读完成状态的可恢复状态，表示用户准备继续阅读或重新朗读的位置。
_Avoid_: 播放历史、已读进度

**朗读完成（Playback Completed）**:
阅读材料的最后一句自然播放结束后形成的状态，表示本轮连续朗读已到末尾，不表示用户已完成阅读。
_Avoid_: 阅读完成、已读
