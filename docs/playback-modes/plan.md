# 播放模式与列表续播 Plan

## 实现方案

- 后端保持材料库导入顺序作为隐含朗读列表顺序，`MaterialNavigation` 增加 `first` 和 `last`。
- 前端新增浏览器级播放模式偏好，默认 `sequential`，存储在 `localStorage`。
- 阅读页播放器增加播放模式菜单，放在倍速控制附近。
- 新增前端播放模式 Module，集中计算自然篇末、手动上一篇和手动下一篇的目标阅读材料。
- 自然篇末自动跨材料时，当前阅读材料仍保存朗读完成状态，并由目标阅读材料自己的阅读进度决定起播落点。

## 关键决策

- 当前列表复用阅读页相邻阅读材料顺序，即 `created_at ASC, id ASC`。
- `repeat_one` 只影响自然篇末；手动上一篇/下一篇仍切换相邻阅读材料。
- `sequential` 到最后一个阅读材料后停止。
- `repeat_list` 在两端 wrap；只有一篇时自然篇末重播当前阅读材料。
- 播放模式切换立即影响下一次篇末或按钮操作，但不打断当前句。
