# 朗读会话 Module Plan

## 实现方案

- 新增非 React 的朗读会话 Module，构造时接收阅读材料句子、初始阅读进度、朗读列表导航、初始播放模式和外部 adapters。
- 朗读会话通过只读 snapshot 暴露当前句、句内位置、播放状态、播放倍速、朗读完成、时间线、错误和音频修复状态。
- 朗读会话通过命令处理播放、暂停、选句、时间跳转、倍速、播放模式、进度重试、音频修复和页面隐藏时保存。
- 现有音频准备、时间线和播放模式 Module 作为朗读会话的 implementation，由朗读会话统一协调。
- 浏览器 audio、材料库音频与进度请求、播放模式偏好和路由跳转通过 adapters 注入；测试使用内存 adapters。
- `ReaderPage` 订阅 snapshot，并仅把当前句变化映射为滚动等 DOM 行为。切换阅读材料时先保存当前进度，再销毁旧会话。

## 关键决策

- 使用非 React Module，而不是自定义 Hook，避免把现有 hooks 原样搬家并继续依赖缺失的 React 交互测试环境。
- 不采用 reducer 与 effect executor，避免为行为不变的重构引入过多事件类型和间接控制流。
- 朗读会话拥有扁平句子序列和音频时长；阅读材料段落结构仍由 `ReaderPage` 用于渲染。
- generation token、进度失败阻塞、后台预取失败静默和音频修复恢复规则保持现有语义。
- 页面滚动、可见句子检测、菜单、阅读偏好和禅模式保留在 `ReaderPage`，避免 DOM 细节跨越 seam。
- 不修改现有未提交的跨平台 TTS 变更，不主动 commit。

## Topic 级验证

- 前端朗读会话测试覆盖恢复与预取、播放与暂停、选句与时间跳转、倍速、句末续播、播放模式、跨材料导航、阅读进度失败重试、音频修复和销毁。
- 现有播放时间线、音频准备、播放模式和阅读页 view model 测试继续通过。
- 运行 `make check`。
- 在浏览器验证阅读材料加载、播放与暂停、选句、时间跳转、倍速和跨材料续播。
