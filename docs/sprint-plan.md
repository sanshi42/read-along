# Read Along Sprint Plan

最后更新：2026-06-07

## 使用方式

这个文档是 MVP 开发过程中的滚动 Sprint 计划。每个 Sprint 开始前更新范围和风险；结束后更新完成情况、验收结果、遗留问题和 backlog 变更。

关联文档：

- `docs/mvp-scope.md`：MVP 做什么、不做什么。
- `docs/product-backlog.md`：用户故事、优先级、状态和依赖。
- `docs/tech-design.md`：最小技术方案。

## Sprint 状态

- `Planned`：计划完成，尚未开始。
- `Active`：正在执行。
- `Review`：实现完成，正在验收。
- `Done`：验收通过。
- `Adjusted`：开发中发生范围调整，需同步 backlog。

## Sprint 0：项目准备

状态：`Planned`

### 目标

建立可持续开发的项目基线，避免后续 Sprint 被依赖安装、启动方式或目录结构卡住。

### 范围

- 确认 Python 依赖管理仍使用 `uv`。
- 确认后端命令形态：`uv run --no-editable read-along serve`。
- 确认前端目录：`web`。
- 确认数据目录默认值：`~/.local/share/read-along/`。
- 确认本地服务默认绑定 `127.0.0.1:8765`。
- 确认 macOS `say` 可用性检测方式。

### 不做

- 不实现业务功能。
- 不做完整 UI。
- 不接入 Scrapling 或 PDF 解析。

### 完成标准

- 新增依赖清单明确。
- README 或开发说明包含本地启动方式。
- 空服务可启动并返回健康检查。

### 风险

- 前端工具链需要额外 Node 环境。
- 后端开发命令统一使用 `uv run --no-editable`。

## Sprint 1：本地材料库与阅读基础

状态：`Done`

### 目标

完成本地材料库、结构化正文和基础阅读页。Sprint 结束时，不需要朗读，但必须能导入文本型 PDF 并阅读。

### Backlog

- MVP-001：本地 FastAPI 服务。
- MVP-002：本地存储。
- MVP-003：数据模型。
- MVP-004：PDF 导入。
- MVP-005：文本结构化。
- MVP-006：前端骨架。
- MVP-007：阅读页。

### 进入条件

- `docs/tech-design.md` 中的数据模型和 API 设计无阻塞争议。
- Python 和前端依赖安装方案明确。

### 实施重点

- 建立 `src/read_along/` 后端模块。
- 初始化 SQLite schema。
- 实现材料、段落、句子、进度的 repository。
- 用 PyMuPDF 提取文本型 PDF。
- 实现段落和句子切分。
- 搭建 React 书架页和阅读页。

### 不做

- 不做网页导入。
- 不做 TTS。
- 不做播放器。
- 不做句子高亮同步。

### 验收演示

1. 启动后端和前端。
2. 上传一个文本型 PDF。
3. 导入完成后材料出现在书架。
4. 打开材料，正文按段落和句子展示。
5. 重启后端后材料仍存在。

### 完成标准

- Sprint 1 Backlog 全部达到 `Done` 或明确拆分新故事。
- 至少有后端单元测试覆盖 PDF 文本提取和句子切分。
- 阅读页没有明显布局阻塞。

### 风险

- PDF 提取质量可能受版式影响。
- 句子切分规则需要在中文材料上校正。

## Sprint 2：网页导入与得到单篇支持

状态：`Active`

### 目标

完成单篇网页导入，覆盖公开网页和用户已授权访问的得到单篇课程页。

### Backlog

- MVP-013：网页导入。
- MVP-014：得到单篇导入。
- MVP-015：重复导入。
- MVP-018：导入任务状态。
- MVP-019：错误提示。

### 进入条件

- Sprint 1 已完成本地材料模型和阅读页。
- 导入任务状态表或等价机制已可用。
- 已确认目标得到单篇课程页需要通过专用 Chrome 会话桥接。

### 实施重点

- 接入 Scrapling 作为公开网页导入入口。
- 实现 URL 导入任务 API。
- 实现 `auto` 和 `chrome` 两种 URL 导入模式。
- 复用现有 Chrome 可见文本读取能力，迁移到 Read Along 导入适配层。
- 扩展正文清洗规则，去除导航、评论区、按钮和页脚噪声。
- 根据 URL 或 `content_hash` 处理重复导入。

### 不做

- 不做课程目录批量导入。
- 不自动翻页。
- 不保存 Cookie、账号密码或导出凭据。
- 不绕过登录或付费权限。

### 验收演示

1. 输入一个公开网页 URL，导入为材料。
2. 输入一个已授权得到单篇课程 URL，使用 Chrome 会话模式导入。
3. 导入任务有状态反馈。
4. 导入后的正文能在阅读页打开。
5. 重复导入相同来源或相同结构化正文不生成重复材料。

### 完成标准

- 公开网页和得到单篇课程页均能通过手动验收。
- 明显评论区和导航噪声不进入正文。
- 抓取失败时有可理解的错误提示。

### 风险

- Scrapling 对动态页面的行为可能需要根据实际 API 调整。
- 得到页面结构变化会影响清洗规则。
- Chrome 会话桥接需要用户手动登录和授权。

## Sprint 3：朗读、高亮和进度

状态：`Planned`

### 目标

完成可用的朗读体验：句子级音频缓存、播放器、当前句高亮、倍速和断点续读。

### Backlog

- MVP-008：TTS 适配器。
- MVP-009：音频缓存。
- MVP-010：播放器。
- MVP-011：句子高亮。
- MVP-012：进度保存。
- MVP-017：倍速播放。

### 进入条件

- Sprint 1 已完成稳定 `sentence_id`。
- 阅读页能定位到每个句子 DOM。
- 后端可按材料和句子查询结构化正文。

### 实施重点

- 实现 TTS 适配器接口。
- 实现 macOS `say` 单句音频生成。
- 将音频缓存到 `~/.local/share/read-along/audio/<material_id>/`。
- 实现音频文件 API。
- 前端使用 `<audio>` 逐句播放。
- 播放句子时同步高亮。
- 保存当前句和倍速。

### 不做

- 不接入 Piper/Kokoro/Coqui。
- 不做逐字或逐词高亮。
- 不做 LLM 改写口播稿。
- 不做复杂音频队列或流式 TTS。

### 验收演示

1. 打开已导入材料。
2. 点击播放，从当前句开始朗读。
3. 当前句被高亮。
4. 一句结束后自动播放下一句并移动高亮。
5. 暂停、上一句、下一句可用。
6. 切换倍速后播放正常。
7. 刷新页面后恢复最近句子。

### 完成标准

- 至少一个 PDF 材料和一个网页材料可完成朗读。
- 单句 TTS 失败不会阻断整篇材料。
- `say` 不可用时有明确错误。

### 风险

- AIFF 在浏览器中的兼容性可能不稳定；如遇问题，切换为 WAVE。
- 长文一次性生成全部音频可能耗时，必要时改成按需生成。

## Sprint 4：体验打磨与 MVP 验收

状态：`Planned`

### 目标

补齐 MVP 可用性和验收闭环，确保第一版能稳定支持日常学习。

### Backlog

- MVP-016：阅读设置。
- MVP-020：删除材料。
- MVP-021：E2E 验收样例。
- MVP-022：音色设置，时间允许。
- MVP-023：音频重试，时间允许。
- MVP-024：阅读定位，时间允许。
- MVP-025：基础来源回查，时间允许。

### 进入条件

- Sprint 1 至 Sprint 3 的 P0 故事已完成或有明确替代方案。
- 核心导入、阅读、朗读、进度闭环可演示。

### 实施重点

- 字号、行距、明暗主题。
- 材料删除和缓存清理。
- 错误和空状态补齐。
- 手动验收脚本。
- 用目标得到课程页、公开网页、文本型 PDF 做端到端验证。

### 不做

- 不做高级学习功能。
- 不做多端同步。
- 不做公网部署。
- 不做扫描版 PDF OCR。

### 验收演示

1. 导入公开网页并朗读。
2. 导入文本型 PDF 并朗读。
3. 使用 Chrome 会话导入得到单篇课程并朗读。
4. 调整阅读设置后刷新页面，设置仍保留。
5. 删除材料后，书架和缓存状态正确。
6. 执行 MVP 手动验收清单。

### 完成标准

- `docs/mvp-scope.md` 的 MVP 验收标准全部通过。
- `docs/product-backlog.md` 中 P0 故事全部 `Done`。
- P1 故事完成或明确说明延期原因。
- 遗留 P2/Future 项回写 backlog。

### 风险

- P2 项可能挤占 MVP 验收时间，必须严格按剩余时间选择。
- 阅读体验打磨容易扩大范围，需要以 scope 文档为准。

## 持续更新规则

- 每个 Sprint 开始前：
  - 更新 Sprint 状态为 `Active`。
  - 确认进入条件。
  - 从 `docs/product-backlog.md` 同步故事状态。
- 每个 Sprint 结束后：
  - 更新完成故事列表。
  - 记录未完成故事和原因。
  - 新增发现的 bug 或范围变更。
  - 将故事状态回写到 `docs/product-backlog.md`。
- 如果开发中发现故事过大：
  - 拆出新故事。
  - 原故事保留目标不变。
  - 新故事补充依赖和优先级。
- 如果需求超出 `docs/mvp-scope.md`：
  - 默认放入 Deferred。
  - 只有明确更新 scope 后才能进入 Sprint。

## 当前风险登记

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| Scrapling 动态页面行为不确定 | 网页导入可能延期 | 先支持公开网页；得到页优先走 Chrome 会话桥接。 |
| Chrome 会话授权流程不稳定 | 得到单篇导入体验受影响 | 保留清晰错误提示和手动操作说明。 |
| `say` 输出格式浏览器兼容性 | 播放器可能无法播放 AIFF | 优先验证 AIFF；失败则改用 WAVE。 |
| 长文音频生成耗时 | 用户等待时间长 | Sprint 3 中评估按需生成或后台进度显示。 |
| PDF 文本提取质量参差 | 阅读正文质量不稳定 | MVP 只承诺文本型 PDF，失败时给明确提示。 |

## 当前决策

- Sprint 数量：5 个，其中 Sprint 0 为准备 Sprint，Sprint 1-4 为 MVP 实现。
- 每个 Sprint 以可演示成果结束。
- P0 故事必须完成才能宣称 MVP 完成。
- P2 项只在 Sprint 4 剩余时间允许时进入。
