# 朗读可靠性修复 Proposal

## Goal

修复部分成对标点导致正文在本地 TTS 音频中被跳过的问题，并减少连续朗读时句与句之间的可感知停顿，使当前朗读体验更忠实、更连贯。

## Boundary

### In

- macOS `say` TTS 适配层的输入归一化。
- 与成对标点包裹正文相关的回归测试。
- 朗读音频缓存与当前 TTS 输入的匹配判定，以及当前材料的手动修复入口。
- 阅读页连续朗读时下一句音频的浏览器端准备与衔接。
- 保持结构化正文、正文哈希、API 返回文本和阅读进度语义不变。

### Out

- 更换 TTS 引擎或引入在线语音服务。
- 修改来源导入、正文抽取或材料去重规则。
- 时间轴式播放控制、句内拖动、快退快进或跨材料导航。
