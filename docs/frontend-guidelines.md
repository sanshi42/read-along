# 前端准则

Read Along 的前端服务于长时间阅读和朗读，不是营销页。界面应安静、稳定、可扫描，优先减少阅读干扰。

## 设计原则

- 阅读材料正文是主角，导航、设置和播放器都应让位于正文。
- 按钮优先使用清晰图标和 `aria-label`，陌生图标加 `title`。
- 朗读状态、错误和修复操作要靠近播放器，不分散到页面远处。
- 移动端和窄屏下，控件不能互相遮挡，也不能挡住当前句。
- 禅模式只隐藏非必要界面，不改变阅读材料、阅读进度或播放模式。

## 组件拆分方向

`ReaderPage.tsx` 当前承担过多职责。后续拆分时保持路由和 class name 兼容，优先拆为：

- `ReaderNav`：返回书架、标题上下文、阅读设置入口、禅模式入口。
- `ReadingPreferencesPanel`：字号、行距、主题和朗读修复。
- `ReaderContent`：标题、来源、段落和句子交互。
- `ReaderPlayerBar`：时间线、播放模式、倍速、跳转和上一篇/下一篇。
- `ZenStatusBar`：禅模式朗读状态、全屏提示和错误信息。
- `useReaderChrome`：当前句滚动、固定控件边界和回到当前句。

拆分顺序应以测试保护为前提：先移动纯展示组件，再移动包含 DOM/滚动副作用的 hook。

## 状态边界

- 跨页面阅读偏好保留在 `App.tsx` 和 `readingPreferences.ts`。
- 当前阅读材料的临时朗读状态保留在 `ReaderPlaybackSession`。
- 播放模式是浏览器端偏好，不写入后端阅读进度。
- 句子音频准备和修复逻辑保留在 `readerAudioPreparation.ts` 与 session 适配器之间。

## 样式约束

- 保持现有 CSS 变量和 class name，避免无关视觉重写。
- 新增控件必须有稳定尺寸，不能因 hover、loading 文案或动态图标造成布局跳动。
- 文字不得溢出按钮、卡片或固定工具栏。
- 不用颜色作为唯一状态信号；错误、当前句和播放中状态应同时有形态或文本提示。
- 前端 formatter 当前渐进接入，不要把大规模格式化和功能改动混在同一个 PR。
