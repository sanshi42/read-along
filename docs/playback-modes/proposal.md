# 播放模式与列表续播 Proposal

## Goal

为阅读页提供浏览器级播放模式，让用户在朗读完成后按单曲循环、顺序播放或列表循环继续朗读阅读材料。

目标完成后，阅读页播放器应能切换播放模式；自然朗读到篇末和手动上一篇/下一篇应按当前模式处理。

## Boundary

### In

- 播放模式偏好的读取、保存、无效值回退和 UI 切换。
- 单曲循环、顺序播放、列表循环在自然篇末和手动跨材料导航时的行为。
- 阅读页相邻阅读材料导航扩展为 first/last/previous/next。
- 跨阅读材料自动续播时沿用目标阅读材料的阅读进度。
- 播放模式、朗读列表领域术语补充。

### Out

- 显式播放队列、书架选择列表或自定义排序。
- 后端持久化播放模式或数据库 schema 变更。
- 保留“播完当前阅读材料就停止”作为独立模式。
- 重新设计书架、导入流程或音频生成策略。
