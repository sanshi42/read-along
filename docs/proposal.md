# Read Along MVP 需求文档

## Summary

构建一个 **macOS 优先的个人本地 Web App**，用于导入单篇网页链接或文本型 PDF，将材料清洗成舒适阅读界面，并用本机 TTS 生成句子级音频，实现播放、暂停、倍速、句子高亮和阅读进度保存。

第一版目标材料是单篇得到课程文章链接。系统只处理用户已授权访问的内容；登录/付费页面通过用户手动登录的专用 Chrome 会话桥接访问，不保存账号凭据，不绕过权限，不做批量课程抓取。

## Key Requirements

- 输入支持：
  - 单篇网页 URL。
  - 文本型 PDF。
  - 第一版不支持扫描版 PDF OCR，不支持课程目录批量导入。
- 网页抓取：
  - 使用 Scrapling 作为统一抓取/解析层。
  - 公开网页可直接抓取。
  - 登录或付费网页使用专用 Chrome 会话桥接，让 Scrapling/浏览器自动化读取用户已打开或已登录会话中的页面。
- 阅读器体验：
  - 清洗正文，保留标题、段落、句子结构。
  - 支持字号、行距、明暗主题。
  - 支持播放/暂停、上一句/下一句、倍速。
  - 当前朗读句子高亮。
  - 保存材料阅读进度和播放进度。
- TTS：
  - 第一版采用可插拔双层策略。
  - 默认用 macOS `say` 生成句子级音频文件。
  - 后续预留 Piper/Kokoro/Coqui 等本地神经 TTS 适配器。
  - 大语言模型第一版不改写正文，避免朗读内容失真。
- 存储：
  - 本地私有库保存材料正文、结构化句子、音频缓存、进度。
  - 不提供云同步，不分享内容，不生成可公开分发的课程包。

## Architecture

- 后端：FastAPI。
  - 导入任务 API：提交 URL/PDF，返回导入任务状态。
  - 材料库 API：列出材料、读取正文结构、更新阅读进度。
  - TTS API：按句子生成/查询音频缓存。
  - 播放资源 API：返回句子音频文件。
- 前端：React。
  - 书架页：展示已导入材料和进度。
  - 阅读页：正文阅读、句子高亮、播放器控制。
  - 导入页：输入 URL 或上传 PDF。
- 内容处理流水线：
  - URL/PDF 输入。
  - 抽取标题、正文、段落。
  - 句子切分，生成稳定 sentence_id。
  - 保存材料结构。
  - 后台按句生成音频。
  - 前端逐句播放并同步高亮。
- 音频策略：
  - 每个句子生成一个 m4a 或 wav 文件。
  - 前端按 sentence_id 顺序播放。
  - 播放结束后自动切换下一句。
  - 倍速优先由浏览器 audio playbackRate 实现。

## Acceptance Criteria

- 输入目标得到文章 URL 后，系统能导入正文并生成可阅读页面。
- 页面正文不包含明显导航、评论区、按钮文案等噪声。
- 点击播放后，能从当前句开始朗读并高亮当前句。
- 暂停后刷新页面，能恢复到最近阅读位置。
- 文本型 PDF 能导入并按段落/句子阅读。
- 重复导入同一 URL 不重复创建材料，或提示已存在。
- 无网络模型依赖；断网时已导入材料和已生成音频仍可阅读播放。
- 不保存 Cookie、账号密码或导出的浏览器凭据。

## Test Plan

- 单篇得到课程页：
  - 导入成功。
  - 正文清洗后不混入评论和页面导航。
  - 句子切分数量合理。
  - 句子级音频可逐句播放。
  - 高亮与当前播放句一致。
- 普通公开网页：
  - Scrapling 可抓取并抽取主要正文。
  - 页面结构变化时不会直接崩溃，失败时给出可理解错误。
- 文本型 PDF：
  - 上传后能提取文本。
  - 多页内容顺序正确。
  - 段落和句子切分可读。
- 本地库：
  - 删除材料会删除对应音频缓存。
  - 重启服务后进度仍存在。
- TTS：
  - `say` 不可用时提示安装/平台限制。
  - 某句生成失败时不阻断整篇材料，可重试该句。

## Assumptions

- 第一版只支持 macOS。
- 第一版是个人本地工具，不面向多人或公网部署。
- 第一版只做单篇导入，不做整门课程批量抓取。
- 第一版不做 OCR。
- 第一版不使用 LLM 改写正文。
- 用户只导入自己有权访问和学习的材料。
- Scrapling 作为抓取层的依据：官方文档称其为 Python 3.10+ 的自适应 Web 抓取框架，支持普通抓取、动态页面和浏览器相关能力。参考：https://scrapling.homes/ 和 https://scrapling.readthedocs.io/
