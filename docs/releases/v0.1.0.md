# Read Along v0.1.0 本地可用版

发布日期：2026-06-22

## 版本定位

`v0.1.0` 是面向本人本机长期使用的本地可用版。它封存当前 MVP 核心闭环：把用户可访问的单篇网页或文本型 PDF 转成可边听边读的本地阅读材料，并支持句子级朗读、同步高亮和断点续读。

这个版本不是对外安装试用版；不承诺安装包、多端同步、账号系统、公网部署或跨平台分发体验。

## 核心能力

- 导入单篇网页和文本型 PDF，保存为材料库中的阅读材料。
- 使用 macOS `say` 生成句子级音频，并复用本地音频缓存。
- 在阅读页播放、暂停、快退、前进、调节倍速和切换播放模式。
- 保存阅读进度、朗读位置、朗读完成状态和阅读偏好。
- 支持时间式朗读进度、跨材料导航和列表续播。
- 支持禅模式，在低干扰界面中阅读、查看朗读状态并通过快捷键进入或退出。

## 明确边界

- 不保存账号密码、Cookie 或导出的浏览器凭据。
- 不绕过登录或付费权限，不做批量课程抓取。
- 不支持 OCR、笔记、划线、全文搜索、LLM 总结或改写。
- 不提供独立安装包、自动更新、多端同步、账号系统或公网部署能力。
- 得到来源适配器是首个来源适配器，不扩大为通用课程平台边界。

## 数据兼容注意事项

- 默认数据目录为 `~/.local/share/read-along`，可通过 `READ_ALONG_HOME` 指向其他目录。
- 应用只接受当前真实六表结构；未知结构、半迁移结构和任意更旧历史结构会保持文件不变并拒绝启动。
- time-navigation 之前的上一版真实六表结构是唯一窄迁移例外，迁移前会备份旧数据库。
- 如果封版后需要尝试新功能，优先使用临时 `READ_ALONG_HOME`，避免影响长期使用的数据目录。

## 封版验收

- 自动检查：`make check` 通过，包括 Ruff、format check、Pyrefly、pytest、前端 node test 和前端生产构建。
- API 烟测：使用临时 `READ_ALONG_HOME` 启动服务，`GET /api/health` 返回 `{"status":"ok","service":"read-along"}`。
- 浏览器验收：完成书架、阅读页、播放控制、阅读设置、播放模式、倍速菜单、禅模式按钮、`Z` 切换、`Escape` 退出和全屏失败降级验证。
- 窄屏验收：在 390x844 视口验证禅模式浅色和深色主题，正文无横向溢出，退出按钮可用，底部状态条不遮挡正文。
