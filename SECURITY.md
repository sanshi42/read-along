# 安全策略

## 威胁模型

Read Along 是本地优先的个人阅读和朗读工具。它处理用户提供的网页 URL、文本型 PDF、本地音频缓存、阅读进度和可选 TTS 服务配置。

### 需要保护的资产

| 资产 | 说明 |
| --- | --- |
| 阅读材料正文 | 从网页或文本型 PDF 清洗得到的结构化正文 |
| 本地源文件 | 导入 PDF 时保存在本机的源文件副本 |
| 句子音频缓存 | TTS 后端生成的本地音频文件和缓存元数据 |
| 阅读进度 | 当前句、句内位置、播放倍速和朗读完成状态 |
| TTS 配置 | 可选在线 TTS 后端的 URL、模型名、API key 或其他凭据 |

### 主要攻击面

- 恶意或异常 PDF 文件。
- 恶意网页内容或不可信来源 URL。
- 可选在线 TTS 后端收到的句子原文。
- 本地音频缓存路径、符号链接和文件系统权限。
- Python、npm 和 GitHub Actions 供应链。

### 当前缓解措施

- 默认数据目录在本机，且 `.env`、缓存、构建产物和模型文件不应提交。
- 不保存账号密码、Cookie 或浏览器凭据。
- 不绕过登录或付费权限，不做批量课程抓取。
- 句子音频缓存路径会验证材料 ID、句子 ID、格式和符号链接逃逸。
- 数据库启动只接受当前已知 schema 或明确记录的受限兼容例外。
- 依赖由 `uv.lock` 和 `web/package-lock.json` 锁定，CI 会运行 CodeQL、Scorecard 和 Dependabot 更新检查。

## 不在范围内

- 用户操作系统或浏览器自身漏洞。
- 用户主动配置的不可信在线 TTS 服务。
- 第三方网站的访问授权、账号安全或付费内容权益。
- OCR、批量抓取、登录态导出和浏览器凭据复用。

## 支持版本

项目仍处于早期版本，安全修复只面向最新 `main` 分支和最新发布版本。

## 报告漏洞

请不要在公开 issue 中披露敏感漏洞细节。优先使用 GitHub 私密安全报告：

<https://github.com/sanshi42/read-along/security/advisories/new>

报告中请包含：

- 问题描述和影响范围。
- 复现步骤、最小样例或测试文件。
- 运行平台、Read Along 版本和相关配置。
- 你认为可行的缓解或修复建议。

维护者会尽量在 3 个工作日内确认收到报告，并在确认漏洞后协调修复和披露。
