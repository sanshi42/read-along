# 朗读会话 Module Proposal

## Goal

加深阅读页的朗读会话 Module，让当前句、音频生命周期、阅读进度保存、播放模式和跨阅读材料续播集中在一个可直接测试的 Interface 后面。

目标完成后，`ReaderPage` 只负责材料加载、DOM 渲染、滚动跟随、菜单、阅读偏好和禅模式；朗读行为保持不变。

## Boundary

### In

- 建立非 React 的朗读会话 Module，以 snapshot 和命令作为阅读页使用的 Interface。
- 将音频准备与播放、阅读进度节流与失败重试、时间线跳转、播放倍速、播放模式和跨材料续播纳入朗读会话。
- 通过浏览器音频、材料库请求和路由跳转 adapters 隔离外部能力。
- 使用内存 adapters 测试真实朗读事件序列。
- 调整 `ReaderPage` 接入朗读会话，并删除被新 Module 吸收的协调状态和回调。

### Out

- 改变任何朗读规则、错误文案、播放模式语义或跨材料导航行为。
- 修改后端、数据库、样式、产品文案或 TTS 实现。
- 引入 React 测试框架或其他新依赖。
- 重构滚动跟随、阅读偏好、禅模式和菜单交互。
